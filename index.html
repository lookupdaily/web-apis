The Document Object Model (DOM)

The Document Object Model(DOM) connects web pages to scripts or programming languages by representing the structure of a document (e.g. HTML) in memory. It is one of the most used web APIs in web development as it enables us to create different types of interactivity with the webpage.

The DOM represents a document with a logical tree. Each branch of the tree ends in a node, which contains objects. By referencing these objects, we can gain access to the tree, and change the document's structure or content.

Full Screen

The Fullscreen API can be used to place a given element into full-screen mode, as well as to detect when the browser enters or exits full-screen mode.

Uses for full screen:

  • Video content
  • Image carousel
  • Article or book

Web Audio API

The Web Audio API complements the media element, allowing complex audio processing and audio controls.

Uses for Web Audio API:

  • Custom styling for audio player (least friendly for accessibility)
  • Visualisations
  • Audio processing/effects

Fetch API

The Fetch API provides an interface for fetching data or resources from different APIs or from a connected server.

This API provides an alternative to XMLHttpRequest used in AJAX programming, but provides a more powerful and flexible feature set.

Web Storage API

The Web Storage API can be used to store small amounts of data without connecting to a server, useful when creating Progressive Web Apps.

The use of this API is synchronous, so the Cache interface and IndexedDB API are more suitable for loading files offline, or for storing more complex data.
